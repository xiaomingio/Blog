Codex 官方手机客户端终于来了

发表于 2026-05-15 分类于 开发技术 阅读次数： Valine：

以前手机上用 Codex 很别扭

在用 Codex 写代码时，经常会遇到需要 Approve，或者 Agent 长时间运行、想看结果的情况。但有时候人不在电脑前，手机上又没办法操作。虽然现在市面上也有一些工具可以实现类似的效果，但都不是特别方便。

比如 HappyCode，我实际测试了一下，Claude 的体验会比较好，可以在手机和电脑上无缝切换。但 Codex 的体验就没那么好了。如果想在手机上操作，电脑上就只能看，不能同时操作，而且电脑上只能用命令行版本。虽然它还有一个网页模式，但我实际测试了几次，都没成功连上。

而 Claude 已经推出了官方手机版 App，但 Codex 一直没有推出来。

今天我发现 Codex 已经能在手机上用了。它其实不是独立 App，而是直接集成在 ChatGPT App 里。

实际接入很简单

我用自己的免费版 ChatGPT 账号测试了一下，还是挺好用的。

首先要安装或升级到最新版 Codex Mac 版，手机上的 ChatGPT App 也要升级到最新版本。Windows暂时还不支持。

Mac 版安装好之后，就会看到这个界面。在 Mac 上登录 Codex 官方账号。

如果你之前设置过 API Key，需要把环境变量以及配置文件 ~/.codex/config.toml 里的 API Key 清掉。不然 Codex 还是会优先调用 API Key，然后发现这个 Key 和你当前登录的账号对不上，最后调用失败。

点击允许之后，会出现一个二维码。实际上这个二维码也不用扫，因为只要手机上登录的是同一个账号，就会直接连上。

在手机上登录同一个账号，然后在菜单里可以看到 Codex 这个选项，点开。

它会打开一个网页，点击授权。

手机上就能看到电脑上之前的项目了。

电脑上也会出现已连接的界面。

敏感操作也能在手机上确认

我实际做了一个测试，直接在手机上发消息，让它帮我把电脑桌面上的一个测试文件夹删掉。删除是一个敏感操作，这个测试比较有代表性。

这时候电脑上会弹出一个 Mac 系统自带的“Codex.app 想访问桌面文件夹中的文件”确认对话框。如果我不点，它就没办法继续。不过还好，这是 Mac 系统的权限控制，只要第一次点过，后面就不需要重复点了。我点了确认。

Codex 在调用了 careful 技能之后，会让我确认是否要删除这个文件夹。而且在电脑和手机上都能看到这个确认弹窗。

我在手机上点了确认，很快它就把这个测试文件夹删掉了，并且按 Codex 一贯的操作，把分析过程那些记录都折叠起来了。

在电脑上也能看到，这个文件夹确实被删掉了。

整个体验很丝滑，但是缺点是必须登录 Codex 官方账号。像我们平时用中转站配置 API Key 的情况，我目前测试是没办法用的，因为手机上必须登录ChatGPT账号，才能看到 Codex 入口。不知道后续会不会有解决办法。
