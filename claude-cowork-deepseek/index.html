不用官方号，Claude Cowork 用国产模型就能做PPT写文章

发表于 2026-05-08 分类于 开发技术 阅读次数： Valine：

背景

前阵子，Claude 的 Cowork 出来了，效果很不错，可以帮我们做 PPT 等各种事情。很多人想用，但这个工具必须用官方账号登录。对国内用户来说太困难了。

后来 Claude 放开了一个限制，允许企业用户配置 Claude 使用云端部署的 Claude 模型。但很多人拿它去调用其他兼容 Anthropic 接口的第三方模型。

再后来，“邪恶的 Anthropic” 在 Claude 新版本又加了限制，网上很多教程都失效了。

当然，这也难不倒广大劳动人民，下面看我的详细介绍。

详细步骤介绍

准备 DeepSeek 的 API Key

打开下面的网址进入 DeepSeek 官网，点 API 开放平台，注册账号。完成实名认证后充值，用微信或者支付宝充 10 块钱就行。然后点击左边的 API Key，创建一个 API Key。

https://www.deepseek.com/

下载 Claude 桌面版应用

在这个官方网站下载。注意这个网站在某些网络环境下可能访问不了。

https://claude.com/download

进入配置界面

运行 Claude 后，默认会进入登录界面。

直接点击菜单里的“帮助 → 故障排除 → 启用开发者模式”，英文版是“Help → Troubleshooting → Enable Developer mode”。

这样菜单里就会多出一个 Developer Mode。打开“开发者 → 配置第三方推理”，英文版是“Developer → Configure third-party inference”，然后你就能看到配置界面，见后面的图。这个过程中 Claude App 可能会重启，等它重启就行。

这几个步骤，Claude 官方文档里其实也写了。

安装和配置 Claude Cowork 与第三方平台的集成 | Anthropic Help Center

配置第三方模型

按照我下图里的方式配置第三方模型的 Base URL 和 API Key，注意模型需要支持 Anthropic 兼容接口。

最重要的一点是，模型名必须以 claude 开头，这是 Claude App 新版本增加的限制。

如果你用的是 DeepSeek，实测发现直接写 claude 开头，后面随便写一个名字就行。DeepSeek 的服务器不会因为模型名字不对就报错，而是会自动给你分配一个模型。

查看 DeepSeek 后台账单可以看到，默认分配的通常是 deepseek-v4-flash。我这个截图里，因为我也通过别的工具调用了 Pro 模型，所以两个模型都有用量。

进阶用法

模型转发

如果你希望一定用 DeepSeek Pro，而不是 Flash，或者你用的不是 DeepSeek，而是其他兼容 Anthropic 接口的模型，情况就不太一样了。这些模型提供商可能不能靠随便填一个模型名来调用，直接这么配可能会报错。

这时候，一个比较省事的办法，是让 AI 帮你写一个转发小工具。在 Claude 里调用本机这个小工具，模型名还是填 claude 开头，但小工具内部会把请求转发给 DeepSeek 或其他模型。

当然，你也不用费劲自己让 AI 现写。这个东西真写出来，还是要花一点时间。你直接用我这个工具就可以了，我自己测试过，没什么问题。如果碰到 bug，也可以让 AI 帮你分析一下，再给我提 PR。

如果你不知道要怎么安装这个项目，你直接把链接丢给 Claude、Codex、小龙虾等任意Agent工具就可以了。注意，如果安装PM2，在Mac上可能需要你同意增加启动项，具体你交给AI解决就好了。

因为它需要在电脑开机时自动启动，确保你电脑一开机，这个服务就会启动。这样你才能持续使用它的转发功能。

https://github.com/jzj1993/llm-model-forward

配置参考

在 Claude 里配置时，Base URL 写本机地址和对应端口，我这个工具默认的就是 http://127.0.0.1:18787/anthropic。模型名还是以 claude 开头，API Key 随便填，后面会被替换掉。

在我这个工具的管理页面里，填真实的 DeepSeek 模型、Base URL 和 API Key。本地模型名和 Claude App 里的名字保持一致。

你可以在 Claude 里配置多个模型，在我这个工具里也配置多个模型，这样在 Claude 里就可以随意切换了。
